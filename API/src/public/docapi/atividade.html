Status Permitidos

Status Permitidos para Atividades

Atividades só podem ser registradas em chamados com os seguintes status:

ATRIBUIDO EMATENDIMENTO
🤖 Comportamento Automático:

Ao registrar a primeira atividade em um chamado com status ATRIBUIDO, o sistema automaticamente atualiza o status para EMATENDIMENTO.

GET /tecnico/{tecnicoId} 🔑 TÉCNICO* | GESTOR

Lista atividades realizadas por um técnico, com filtro por período.

📥 Parameters

tecnicoId* ID do técnico (path)
dataInicio YYYY-MM-DD
dataFim YYYY-MM-DD

limite Itens por página (default: 10)

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found
📋 Regras de Acesso:
  • TÉCNICO: Pode ver apenas suas próprias atividades
  • GESTOR: Pode ver atividades de técnicos da sua unidade

Endpoints de Manipulação (Apenas Técnicos)

POST /chamado/{chamadoId} 🔧 APENAS TÉCNICO

Registra uma nova atividade em um chamado. Apenas técnicos da equipe responsável.

📥 Request

chamadoId* ID do chamado (path)
AtividadeDescricao* Descrição da atividade realizada 
{
  "AtividadeDescricao": "Realizada análise inicial do problema. Identificado que o cabo de rede está danificado."
}

📤 Responses

201 Created 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found
⚠️ Validações Rigorosas:
  • Técnico deve estar ATIVO
  • Chamado deve estar com status: ATRIBUIDO, EMATENDIMENTO ou ANALISADO
  • Chamado deve ter uma equipe atribuída
  • Técnico deve pertencer à equipe responsável pelo chamado
  • Técnico deve ser da mesma unidade do chamado
PUT /{id} 🔧 APENAS TÉCNICO (autor)

Altera a descrição de uma atividade. Apenas o técnico que criou pode alterar.

📥 Request

id* ID da atividade (path)
AtividadeDescricao* Nova descrição da atividade 
{
  "AtividadeDescricao": "Descrição atualizada da atividade..."
}

📤 Responses

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found
⚠️ Restrições:
  • Apenas o técnico autor da atividade pode alterá-la
  • Técnico deve estar ATIVO
  • Chamado deve estar em andamento (ATRIBUIDO, EMATENDIMENTO, ANALISADO)
  • Não é possível alterar atividades de chamados concluídos ou cancelados
DELETE /{id} 🔧 APENAS TÉCNICO (autor)

Exclui uma atividade. Apenas o técnico que criou pode excluir.

📥 Path Parameters

id* ID da atividade

📤 Responses

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found 
{
  "message": "Atividade excluída com sucesso"
}
⚠️ Mesmas restrições da alteração:
  • Apenas o técnico autor pode excluir
  • Técnico deve estar ATIVO
  • Chamado deve estar em andamento

Modelo de Dados - Atividade de Chamado

AtividadeId Integer - ID único da atividade
ChamadoId Integer - ID do chamado (FK)
TecnicoId Integer - ID do técnico (FK)
AtividadeDescricao Text - Descrição detalhada da atividade
AtividadeDtRealizacao DateTime - Data e hora da realização

Códigos de Erro Específicos

400 Descrição obrigatória, status do chamado não permite atividades
401 Token não fornecido ou inválido
403 Técnico inativo, não pertence à equipe, unidade diferente, não é autor da atividade
404 Atividade, chamado ou técnico não encontrado
📌 Resumo de Permissões:
  • CRIAR: Técnico ATIVO, da equipe responsável, chamado em andamento
  • ALTERAR: Técnico autor ATIVO, chamado em andamento
  • EXCLUIR: Técnico autor ATIVO, chamado em andamento
  • VISUALIZAR: Pessoa (próprios chamados), Técnico (unidade/equipe), Gestor (unidade)
🔍 Observação: A API não permite criar, alterar ou excluir atividades em chamados já concluídos (CONCLUIDO, CANCELADO, RECUSADO) para manter a integridade do histórico.
