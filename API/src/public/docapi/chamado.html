Endpoints Protegidos (Requer Token JWT)

GET / 🔑 PESSOA | TÉCNICO | GESTOR

Lista chamados com múltiplos filtros. Acesso varia conforme perfil.

📥 Query Parameters

unidadeId Filtrar por unidade
pessoaId Filtrar por pessoa (gestor/admin)
equipeId Filtrar por equipe
tipoSuporteId Filtrar por tipo de suporte
status Separar por vírgula: PENDENTE,ANALISADO
urgencia Separar por vírgula: BAIXA,MEDIA,ALTA,URGENTE
prioridadeMin Valor mínimo (1-10)
prioridadeMax Valor máximo (1-10)
dataInicio YYYY-MM-DD
dataFim YYYY-MM-DD
pagina Número da página (default: 1)
limite Itens por página (default: 10)

📤 Responses

200 OK 400 Bad Request 401 Unauthorized
🔍 Regras de Visualização:
  • PESSOA: Apenas seus próprios chamados
  • TÉCNICO: Chamados da unidade OU da sua equipe
  • GESTOR: Todos chamados da sua unidade
GET /{id} 🔑 PESSOA | TÉCNICO | GESTOR

Busca chamado por ID com todas as atividades e detalhes da equipe.

📥 Path Parameters

id* ID do chamado

📤 Responses

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found
POST / 🔑 PESSOA

Abre um novo chamado. Apenas pessoas podem criar chamados.

📥 Request Body

PessoaId* ID da pessoa (deve ser a logada)
UnidadeId* ID da unidade (deve ser da pessoa)
ChamadoDescricaoInicial* Descrição inicial do problema 
{
  "PessoaId": 1,
  "UnidadeId": 1,
  "ChamadoDescricaoInicial": "Computador não liga"
}

📤 Responses

201 Created 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found
⚠️ Validações:
  • Pessoa deve estar ATIVA
  • Unidade deve estar ATIVA
  • Status inicial: PENDENTE
PUT /{id} 🔑 PESSOA* | GESTOR | TÉCNICO

Altera dados do chamado. Pessoa tem restrições.

📥 Parameters

id* ID do chamado (path)

📤 Responses

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found
📋 Regras de Permissão:
  • PESSOA: Pode alterar apenas título e descrição, e apenas se chamado ainda estiver PENDENTE
  • TÉCNICO: Pode alterar campos conforme permissão da equipe
  • GESTOR: Pode alterar todos os campos (tipos de suporte, equipe, prioridade, urgência, status)
PATCH /{id}/status 🔑 GESTOR | TÉCNICO

Altera status do chamado seguindo fluxo válido.

📥 Request Body

ChamadoStatus* PENDENTE, ANALISADO, ATRIBUIDO, EMATENDIMENTO, CONCLUIDO, CANCELADO, RECUSADO 
{
  "ChamadoStatus": "EMATENDIMENTO"
}

📤 Responses

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found
⚠️ Restrições:
  • TÉCNICO: Não pode CANCELAR ou RECUSAR chamados
  • GESTOR: Pode realizar todas as transições
  • Status CONCLUIDO, CANCELADO e RECUSADO adicionam data de encerramento automaticamente
PATCH /{id}/atribuir-equipe 🔑 GESTOR

Atribui uma equipe ao chamado. Se chamado estiver PENDENTE, passa para ANALISADO.

📥 Request Body

EquipeId* ID da equipe (deve ser da mesma unidade) 
{
  "EquipeId": 3
}

📤 Responses

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found

Modelo de Dados - Chamado

ChamadoId Integer - ID único do chamado
PessoaId Integer - ID da pessoa (FK)
UnidadeId Integer - ID da unidade (FK)
TipSupId Integer - ID do tipo de suporte (FK)
EquipeId Integer - ID da equipe responsável (FK)
ChamadoTitulo String - Título do chamado
ChamadoDescricaoInicial Text - Descrição inicial do problema
ChamadoDescricaoFormatada Text - Descrição formatada
ChamadoPrioridade Integer - 1 a 10
ChamadoUrgencia Enum - BAIXA, MEDIA, ALTA, URGENTE
ChamadoStatus Enum - PENDENTE, ANALISADO, ATRIBUIDO, EMATENDIMENTO, CONCLUIDO, CANCELADO, RECUSADO
ChamadoDtAbertura DateTime - Data de abertura
ChamadoDtEncerramento DateTime - Data de encerramento

Códigos de Erro Específicos

400 Campos obrigatórios, prioridade inválida, urgência inválida, status inválido
401 Token não fornecido ou inválido
403 Pessoa inativa, unidade inativa, permissão negada, transição de status inválida
404 Chamado, pessoa, unidade, equipe ou tipo de suporte não encontrado
🔍 Observação Importante: A API não possui rota de DELETE para chamados, conforme design do sistema. O encerramento é feito via alteração de status para CONCLUIDO, CANCELADO ou RECUSADO.
