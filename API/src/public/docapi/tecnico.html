Autenticação (Pública)

POST /login

Realiza login do técnico utilizando usuário e senha.

📥 Request Body

TecnicoUsuario* Nome de usuário do técnico (UPPERCASE)
TecnicoSenha* Senha do técnico 
{
  "TecnicoUsuario": "TECNICO01",
  "TecnicoSenha": "minhasenha123"
}

📤 Responses

200 OK 400 Bad Request 403 Forbidden 500 Error 
{
  "message": "Login realizado com sucesso",
  "data": {
    "usuario": {
      "TecnicoId": 1,
      "TecnicoNome": "João Técnico",
      "TecnicoEmail": "joao.tecnico@email.com",
      "Unidade": {
        "UnidadeNome": "Unidade Centro"
      },
      "Departamento": {
        "DepartamentoNome": "TI"
      }
    },
    "token": "eyJhbGc...",
    "tipo": "TECNICO"
  }
}
⚠️ Validações:
  • Técnico deve estar ATIVO
  • Unidade deve estar ATIVA
  • Departamento deve estar ATIVO
  • Senha validada com pepper específico

Endpoints Protegidos (Requer Token JWT)

GET / 🔑 GESTOR | TÉCNICO

Lista técnicos com filtros. Gestores veem todos da unidade, técnicos veem apenas seus dados.

📥 Query Parameters

unidadeId Filtrar por unidade (apenas gestor)
departamentoId Filtrar por departamento (apenas gestor)
status ATIVO, INATIVO, BLOQUEADO
nome Busca por nome
pagina Número da página (default: 1)
limite Itens por página (default: 10)

📤 Responses

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden

📥 Path Parameters

id* ID do técnico

📤 Responses

POST / 🔑 GESTOR (mesma unidade)

Cadastra novo técnico. Apenas gestores da mesma unidade.

📥 Request Body

DepartamentoId* ID do departamento
UnidadeId* ID da unidade
TecnicoNome* Nome completo
TecnicoCPF* CPF (único)
TecnicoUsuario* Nome de usuário (único)
TecnicoSenha* Mínimo 6 caracteres
TecnicoEmail E-mail válido
TecnicoTelefone Telefone para contato
TecnicoStatus ATIVO, INATIVO, BLOQUEADO (default: ATIVO) 
{
  "DepartamentoId": 1,
  "UnidadeId": 1,
  "TecnicoNome": "Carlos Técnico",
  "TecnicoEmail": "carlos.tecnico@email.com",
  "TecnicoTelefone": "11988887777",
  "TecnicoCPF": "12345678901",
  "TecnicoUsuario": "CTECNICO",
  "TecnicoSenha": "senha123"
}

📤 Responses

201 Created 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found 409 Conflict
⚠️ Validações:
  • Gestor deve estar ATIVO
  • Unidade deve estar ATIVA
  • Departamento deve estar ATIVO e pertencer à unidade
  • CPF e Usuário não podem ser duplicados
PUT /{id} 🔑 GESTOR | PRÓPRIO TÉCNICO

Altera dados do técnico. Gestor pode alterar tudo, técnico só dados básicos.

📥 Parameters

id* ID do técnico (path)

📤 Responses

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found 409 Conflict
📋 Regras de Permissão:
  • TÉCNICO: Pode alterar nome, email, telefone e senha. Não pode alterar status, unidade, departamento, CPF ou usuário.
  • GESTOR: Pode alterar todos os campos, desde que seja da mesma unidade e esteja ativo.
PATCH /{id}/status 🔑 GESTOR (mesma unidade)

Altera apenas o status do técnico. Apenas gestores.

📥 Request Body

TecnicoStatus* ATIVO, INATIVO, BLOQUEADO 
{
  "TecnicoStatus": "BLOQUEADO"
}

📤 Responses

200 OK 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found
⚠️ Restrições:
  • Não é possível INATIVAR/BLOQUEAR técnico em equipes ativas
  • Não é possível INATIVAR/BLOQUEAR técnico com chamados em andamento
  • Gestor deve estar ATIVO
  • Unidade deve estar ATIVA

Modelo de Dados - Técnico

TecnicoId Integer - ID único do técnico
DepartamentoId Integer - ID do departamento (FK)
UnidadeId Integer - ID da unidade (FK)
TecnicoNome String - Nome completo
TecnicoEmail String - E-mail (único)
TecnicoTelefone String - Telefone
TecnicoCPF String - CPF (único)
TecnicoUsuario String - Nome de usuário (único)
TecnicoStatus Enum - ATIVO, INATIVO, BLOQUEADO
TecnicoDtCadastro DateTime - Data de cadastro

Códigos de Erro Específicos

400 Campo obrigatório não preenchido, senha muito curta, status inválido
401 Token não fornecido ou inválido
403 Técnico inativo, unidade inativa, departamento inativo, permissão negada
404 Técnico, unidade ou departamento não encontrado
409 CPF ou nome de usuário já cadastrado
🔍 Observação Importante: A API não possui rota de DELETE para técnicos, conforme design do sistema. Para desabilitar um técnico, utilize o PATCH de status.
